loadDB() ignores the .error field on all seven Supabase selects. Supabase resolves a failed query
as {data: null, error} instead of throwing, so `rooms.data || []` turns a connection failure into
an empty array and the app renders as a real but completely empty database — no rooms, no bills,
no payment history, and settings silently reset to defaults. Nothing tells the user the read
failed. Worst case they re-enter data, each new row gets a fresh crypto.randomUUID(), nothing
collides with the real rows still in Supabase, and the database ends up duplicated.

Found during the first AI Dev OS audit of index.html. via: manual (entered by Claude during
pipeline setup, not captured from Telegram).
